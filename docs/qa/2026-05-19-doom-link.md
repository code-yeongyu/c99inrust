# Doom Link And Run QA

Public Doom checkout: `id-Software/DOOM` at `/tmp/c99inrust-doom-src`.

This QA evidence started as compile/link proof and now also records the first
non-interactive manual run smoke with a legal IWAD under Xvfb. It proves the
binary survives startup and map-load smoke until the scripted timeout. Visible
window and keyboard-control playability remain separate gates.

## Assembly Scan

The scan ran in tmux without `tmux kill-server`:

```text
tmux_session=c99inrust-doom-link-1779165229
scan=/tmp/c99inrust-doom-link-1779165229.txt
asm_dir=/tmp/c99inrust-doom-link-1779165229-asm
ok=62
fail=0
```

The command compiled all official `linuxdoom-1.10` C translation units with the
upstream Linux build defines and the x86_64 Linux target:

```bash
target/debug/c99inrust compile -S \
  --target x86_64-unknown-linux-gnu \
  -D NORMALUNIX -D LINUX \
  -I /tmp/c99inrust-doom-src/linuxdoom-1.10 \
  /tmp/c99inrust-doom-src/linuxdoom-1.10/<unit>.c \
  -o /tmp/c99inrust-doom-link-1779165229-asm/<unit>.s
```

## Link Probe

The generated assembly linked successfully in an amd64 Ubuntu container:

```bash
docker run --rm --platform linux/amd64 \
  -v /tmp/c99inrust-doom-link-1779165229-asm:/asm:ro \
  -v /tmp/c99inrust-doom-link-1779165229-out:/out \
  ubuntu:24.04 bash -lc \
  'apt-get update >/tmp/apt.log && apt-get install -y --no-install-recommends gcc libc6-dev libx11-dev libxext-dev >/tmp/apt-install.log && gcc -o /out/linuxdoom-c99inrust /asm/*.s -lm -lX11 -lXext'
```

Result:

```text
binary=/tmp/c99inrust-doom-link-1779165229-out/linuxdoom-c99inrust
ELF 64-bit LSB pie executable, x86-64, dynamically linked
```

All generated assembly files include `.note.GNU-stack`, so the linker did not
emit an executable-stack warning.

## Xvfb Run Smoke

The current full smoke was run in tmux without `tmux kill-server`:

```text
tmux_session=c99inrust-doom-qa-1779255203
script=/tmp/c99inrust-doom-qa-1779255203.sh
out=/tmp/c99inrust-doom-qa-1779255203-out
compile_ok=62 compile_fail=0
link_status=0
run_status=124
```

`run_status=124` is the scripted 25 second timeout, not a segfault. `rg` over
the run log found no `Segmentation` line and no Doom `Error:` line. The log
reached:

```text
DOOM Shareware Startup v1.10
W_Init: Init WADfiles.
 adding ./doom1.wad
R_Init: Init DOOM refresh daemon
P_Init: Init Playloop state.
ST_Init: Init status bar.
Using MITSHM extension
shared memory id=0, addr=0xff0c4000
```

Repeat with:

```bash
cargo build
tools/doom-smoke.sh /tmp/c99inrust-doom-src /path/to/doom1.wad /tmp/c99inrust-doom-smoke
```
